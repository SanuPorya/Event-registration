# Event-registration
This website is based on the problem statement of DevFeast 5.0 
# Instructions for The Judges :
1. The Access code for volunteer and admin is "code_yaha_hai". This feature is added to ensure that no attendee can access the volunteer's and admin's dashboard
2.  The Roles in this website are as follows:-
  * Attendee ( These are the normal guests who have to pay on the QR as they will get to choose the events and pay according to their selected events, which will be displayed in their Dashboard. They will also receive a QR, which is to be scanned by the volunteers.)
  * Volunteer ( They will have a scanner at the site to let them scan the QR of the attendee, the info will be stored in the database, which the Admin can monitor )
  * Admin ( they can add, remove, edit events accordingly, as well as they can monitor the number of registrations and the info of which volunteer has done how many scans )
# Features
1. Different Themes based UI for different Roles.
2. Inbuilt scanner for Volunteers
3. Easy Managing Systems for Admin to control the entire event with Ease on their Dashboard.
4. Prevent Multiple registrations with a single set of credentials.
